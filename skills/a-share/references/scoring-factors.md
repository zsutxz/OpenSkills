# 评分因子与权重

本 skill 用**多因子综合打分**从候选池中筛选次日推荐股票。总分 0–100，越高越优先。设计目标：可解释（每只股票都附带分项理由）、可调（权重集中在 `scoring.py` 的 `WEIGHTS` 常量）。

## 总分公式

```
总分 = w_tech · 技术动能 + w_money · 资金面 + w_sector · 板块热度 − w_risk · 风险
```

最后 `clamp` 到 [0, 100]。默认权重（MVP 经验值）：

| 因子 | 权重 | 方向 |
|------|------|------|
| 技术动能 `tech` | 0.40 | 正向 |
| 资金面 `money` | 0.30 | 正向 |
| 板块热度 `sector` | 0.20 | 正向 |
| 风险 `risk` | 0.10 | **负向**（风险越高总分越低） |

> 调参：直接改 `scoring.py: WEIGHTS`。权重无需归一化（公式是线性加权再 clamp）。后续若要做网格搜索优化，可参考 `world-cup-predictor/scripts/grid_search_weights.py` 的做法。

## 因子族定义（每族归一化到 0–100）

### 1. 技术动能 `tech`（基于近 60 日 K 线）

| 条件 | 加分 |
|------|------|
| 均线多头排列：MA5 > MA10 > MA20 | +25 |
| 收盘价站上 MA20 | +15 |
| 收盘价站上 MA60 | +10 |
| MACD：DIF > DEA 或 红柱（hist > 0） | +20 |
| 近 20 日涨幅 ∈ [5%, 25%]（健康上升趋势） | +15 |
| 放量：量比 > 1.2 | +15 |

> 上述各项累加后 clamp 到 [0, 100]。趋势向上且量价配合得分最高；高位滞涨或下行趋势得分低。

### 2. 资金面 `money`（基于主力资金流 `f62`）

| 条件 | 加分 |
|------|------|
| 当日主力净流入 > 0 | 基础 +30 |
| 主力净流入占比 > 3% | +25 |
| 超大单净流入 > 0 | +20 |
| 价格上涨且主力流入（量价齐升） | +25 |

> 北向资金实时不可用（见 `data-sources.md`），故资金面完全由主力/大单驱动。资金流出但股价上涨（量价背离）在此因子低分，并在风险因子中额外扣分。

### 3. 板块热度 `sector`（基于行业板块当日涨幅）

| 板块当日涨幅 | 得分 |
|--------------|------|
| ≥ 3% | 90–100 |
| 1% ~ 3% | 60–90 |
| 0% ~ 1% | 40–60 |
| < 0% | 0–40 |

> 顺风板块（所在行业当日强势）加分，逆风板块扣分。板块映射来自个股所属行业（快照接口的板块字段 / 兜底按名称归类）。

### 4. 风险 `risk`（减分项，0–100，越高风险越大）

| 条件 | 风险分贡献 |
|------|------------|
| 近 5 日涨幅 > 30%（短期透支，防追高接盘） | +40 |
| 近 5 日涨幅 ∈ [20%, 30%] | +20 |
| 当日振幅 > 8% | +20 |
| 量价背离（涨但主力净流出） | +25 |
| 换手率 > 25%（过度投机） | +15 |

> 风险分高的股票即使技术面好看，总分也会被压低，避免推荐在高位接盘。

## 候选池过滤（打分前剔除）

进入打分前，先从全市场快照剔除以下股票（不占候选名额）：

- **ST / *ST / 退市**：名称含 `ST`、`*ST`、`退`
- **当日涨跌停**：涨跌幅接近涨停板（主板 ≥9.8%、双创 ≥19.5%、北交所 ≥29.5%）或 ≤ −9.8%，次日难买入/已无空间
- **次新股**：上市不足 60 个交易日（K 线不足无法算指标，且波动失真）
- **亏损股**：动态市盈率 `f9 < 0`
- **极端换手**：换手率 < 0.3%（僵死）或 > 30%（纯投机）
- **流动性差**：成交额 < 5000 万元
- **价格异常**：最新价 ≤ 0 或为空

过滤后通常剩 100–300 只，再按快照数据粗排（涨幅 + 换手综合）取前 ~120 只进入第 3 层 K 线打分（避免逐只抓全市场）。

## 输出理由

每只推荐股票除总分外，附带 **2–3 条人类可读理由**（如"均线多头排列 + MACD 金叉"、"主力净流入 1.2 亿占成交 5%"、"所属半导体板块涨 3.2%"），由 `scoring.py` 命中条件的描述模板拼接，供报告直接展示。
