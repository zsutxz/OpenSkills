## 何时激活与如何取输入

此 skill 自动激活，无需手动 `/` 调用。识别到翻译意图后，从用户输入中获取待翻译内容：

- **URL**（如 `https://example.com/post`）：用 WebFetch 抓取正文
- **直接粘贴的文本**：直接翻译

中英文表述都能触发——“翻译这篇技术文章”、“把这段英文翻译成中文”、“translate this technical article”、“help me translate this IT documentation” 等。

## 工作流程

1. **获取内容**：URL 用 WebFetch 抓取正文；已粘贴文本则直接使用。
2. **分析文章**：判断类型（技术博客 / 学术论文 / 新闻资讯），提取专业术语建立对照表。
3. **翻译**：按段落与标题结构翻译，遵循下方准则。
4. **校对**：检查技术准确性、表达流畅性、格式完整性。
5. **保存**：写入本地 Markdown 文件（见「输出与保存」）。

## 翻译准则

**准确性优先**
- 核心数据、技术原理、逻辑关系必须与原文一致，不夸大、不臆测。
- 代码块、命令行、API 调用保持原样，不翻译。

**术语规范**
- 使用行业公认译法，首次出现标注英文：`人工智能 (Artificial Intelligence)`，后续用 `AI` 或 `人工智能`。
- 同一术语全文译法保持一致。

**表达风格**
- 像讲故事一样解释概念，避免枯燥学术腔；用自然的中文语序，避免翻译腔。
- 长句适当拆分，保持逻辑链条清晰。

**格式保留**
- 完整保留标题层级、粗体/斜体、有序与无序列表、链接、引用块；翻译图片 alt 文本并保留图片链接。

**文化适配**
- 对欧美特有的技术背景或案例做适当解释，或替换为更易懂的中文对应。
- 需要补充说明处用 `**（注释内容）**` 标注。

## 输出与保存

**文件命名**
- URL 文章：`[网站名]_[文章标题]_zh.md`
- 粘贴文本：`翻译文章_[日期时间]_zh.md`

**文件结构**
```markdown
# [翻译后的标题]

**原文链接**: [原始 URL]
**翻译时间**: [日期]
**文章类型**: [技术博客 / 学术论文 / 新闻资讯]

---

[翻译后的正文，保持完整格式]
```

## 自检清单

保存前逐项核对：

- [ ] 术语译法正确且全文一致；代码 / 命令未被误译
- [ ] 数据与事实与原文一致，逻辑完整
- [ ] 中文表达自然，无翻译腔，长句已拆分
- [ ] Markdown 格式完整：标题层级、列表、链接、代码块、图片正常

## 示例

原文：
```
## Machine Learning in Production

Deploying ML models at scale requires careful consideration of latency, throughput, and monitoring.
```

译文：
```
## 生产环境中的机器学习

大规模部署机器学习模型时，需要仔细权衡延迟、吞吐量和监控等关键因素。
```
